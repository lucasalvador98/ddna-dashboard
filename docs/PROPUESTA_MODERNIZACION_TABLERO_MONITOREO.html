Defensoría de los Derechos de Niñas, Niños y Adolescentes

Modernización del Tablero General de Monitoreo

Propuesta técnica para la mejora del sistema de visualización de datos sobre infancia y adolescencia en la Provincia de Córdoba

Resumen Ejecutivo

Objetivo del proyecto: Desarrollar un sistema de monitoreo y visualización de datos moderno, mantenible y accesible que permita el análisis integral de indicadores de infancia y adolescencia en la Provincia de Córdoba.

El presente documento propone la modernización del actual Tablero General de Monitoreo, reemplazando la dependencia de Power BI por una solución basada en código abierto (Python + Streamlit). Esta transformación permitirá automatizar la actualización de datos, mejorar la presentación visual para audiencias ejecutivas, y crear un repositorio consultable de fuentes para investigadores y ciudadanos.

1. Situación Actual

1.1 Sistema vigente

El Tablero General opera actualmente sobre una arquitectura que combina Power BI (.pbix) con archivos CSV/Excel como fuentes de datos. La actualización es 100% manual: cada vez que una fuente oficial publica nuevos datos, el equipo técnico debe descargar, procesar e integrar manualmente los archivos al tablero.

3.3 Módulos funcionales

  1. Módulo de Gestión de Datos: Carga de archivos, validación, historial de actualizaciones, documentación de fuentes
  2. Módulo de Visualización: Dashboard principal, secciones por tema (Salud, Educación, Pobreza, Seguridad), mapas geográficos
  3. Módulo de Informes: Generación de reportes PDF, exportación de datos, configuración de métricas
  4. Módulo de Consulta de Fuentes: Catálogo de fuentes con metadatos, links a originales, frecuencia de actualización

3.4 Recuperación de datos

Se propone una arquitectura híbrida para la ingestión de los CSV de fuentes oficiales:

  • Descarga programada (Python): Scripts etl_*.py que utilizan requests o la API de datos.gob.ar para obtener los archivos CSV y guardarlos en la carpeta raw/.
  • ETL de transformación: Un pipeline de pandas que normaliza, limpia y genera los datasets finales en processed/, aplicando validaciones de calidad.
  • Almacenamiento intermedio: SQLite (o Supabase/PostgreSQL) para consultas rápidas desde el frontend.
  • Actualización automática: Cron (o GitHub Actions) que ejecuta los scripts cada 24 h o al publicar una nueva versión de la fuente.
  • Fallback manual: Cuando la fuente no cuenta con API, se permite la carga manual del CSV a raw/, disparando el mismo pipeline.

Este flujo permite que el frontend (Vue/React) consuma los datos a través de una API REST (FastAPI) o directamente vía endpoint de Supabase, garantizando que el dashboard siempre muestre la información más reciente sin intervención humana.

6. Conclusión

Esta propuesta representa una evolución significativa en la capacidad de la Defensoría para monitorear y comunicar información sobre Niñez y Adolescencia. El sistema propuesto prioriza:

  • Estética profesional para presentaciones a autoridades
  • Flexibilidad para carga manual o automatizada
  • Sostenibilidad sin dependencia de herramientas proprietarias
  • Transparencia mediante repositorio de fuentes consultable

Este proyecto posiciona a la Defensoría como una institución moderne, con capacidades técnicas avanzadas para el análisis de datos públicos y la generación de conocimiento para la protección de los derechos de niñas, niños y adolescentes.
